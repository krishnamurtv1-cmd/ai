## FitFindr takes a natural-language clothing request and first calls search_listings to find matching secondhand items based on description, size, and price. If a listing is found, the agent stores the selected item in session state, then calls suggest_outfit using the user’s wardrobe to generate a complete outfit idea, followed by create_fit_card to produce a short, shareable caption. If search_listings returns no results, the agent stops early, explains what failed, and suggests how the user can adjust their request instead of calling downstream tools with empty input.

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->

The agent uses a single planning loop that executes tools conditionally based on returned values and updates a shared session state dictionary.

Parse the user’s natural-language query to extract:
description (string)
size (string or None)
max_price (float or None)
Call search_listings(description, size, max_price).
After search_listings returns:
If results is an empty list ([]):
Set session["error"] to a clear, actionable message such as
"No listings matched your request. Try increasing the price limit or removing the size filter."
Do not call any other tools.
Return the session immediately.
If results contains one or more items:
Select the top result: selected_item = results[0]
Store it in state as session["selected_item"]
Call suggest_outfit(selected_item, wardrobe) using the stored item and the user’s wardrobe.
After suggest_outfit returns:
Store the returned string as session["outfit_suggestion"]
Continue even if the wardrobe is empty, since the tool handles that case internally.
Call create_fit_card(outfit_suggestion, selected_item).
After create_fit_card returns:
Store the result as session["fit_card"]
Return the completed session containing:
selected_item
outfit_suggestion
fit_card
(optionally) error if execution stopped early

Write out what a full user interaction looks like from start to finish — tool call by tool call. Use a specific example query.

**Step 1:**
<!-- What does the agent do first? Which tool is called? With what input? -->

The agent first extracts the key constraints from the user’s request:

Item type: vintage graphic tee
Maximum price: $30
Implied size: not specified, so no size filter is applied
Goal: find available items before suggesting an outfit

The agent calls the search_listings tool to retrieve matching items.

Tool call:

  query="vintage graphic tee",
  max_price=30.0
)

The tool returns a list of matching listings sorted by relevance. For example:

“Faded Band Tee” — $22 — Depop — Good condition
“Vintage Skull Graphic Tee” — $28 — Etsy — Fair condition
“90s Tour Tee” — $30 — Grailed — Very good condition

The agent selects the top result as the best match.

**Step 2:**
<!-- What happens next? What was returned from step 1? What tool is called now? -->

With a valid listing selected, the agent now focuses on styling. It considers:

The new item (vintage band tee)

The user’s existing wardrobe (if available)

The agent calls the suggest_outfit tool using the chosen listing and the user’s wardrobe context.
